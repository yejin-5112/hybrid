핵심 역량

  • **프론트엔드 개발**: HTML5, CSS3, JavaScript를 기반으로 사용자 경험(UX)을 고려한 반응형 웹 페이지를 개발할 수 있습니다. React를 활용한 컴포넌트 기반 개발에 익숙하며, 재사용 가능한 코드를 작성하기 위해 노력합니다.
  • **반응형 웹 디자인**: 다양한 디바이스 환경에 최적화된 UI/UX를 구현하며, 미디어 쿼리와 유연한 레이아웃을 활용하여 사용자 접근성을 높입니다.
  • **UI/UX 이해**: 사용자 행동 패턴을 분석하고, 사용성 테스트를 통해 직관적이고 효율적인 인터페이스를 설계합니다. Figma를 활용한 프로토타이핑 경험이 있습니다.
  • **협업 및 소통**: 팀 프로젝트에서 Jira, Git을 활용해 효율적으로 협업하고, 원활한 커뮤니케이션을 통해 프로젝트 목표를 달성했습니다. 코드 리뷰를 통해 동료들과 함께 성장하는 것을 중요하게 생각합니다.

레포트: 반응형 웹 UX 개선

개요

본 레포트는 특정 웹사이트의 모바일 환경 UX(사용자 경험) 문제를 분석하고, 이를 개선하기 위한 방안을 제시 및 적용하여 사용자 만족도를 향상시킨 과정을 담고 있습니다. 사용자 이탈률 감소와 접근성 증대를 목표로 하였습니다.

문제 분석

기존 웹사이트는 모바일 환경에서 다음과 같은 문제점을 안고 있었습니다.

  • 낮은 텍스트 가독성: 작은 글씨 크기와 부적절한 줄 간격으로 정보 습득 어려움.
  • 복잡한 네비게이션: 모바일 화면에 맞지 않는 메뉴 구조로 인한 사용자 혼란.
  • 긴 페이지 로딩 시간: 최적화되지 않은 이미지와 스크립트로 인한 로딩 지연.
  • 데이터 분석 결과: 모바일 환경에서 데스크톱 대비 30% 높은 이탈률과 20% 낮은 페이지당 체류 시간을 기록.

개선 방안 및 적용

레이아웃 최적화

모바일 뷰포트에 맞춰 유연한 그리드 시스템과 Flexbox를 활용하여 콘텐츠의 흐름을 재배치했습니다. 텍스트 크기, 버튼 크기를 터치하기 쉽도록 조정하고, 충분한 여백을 확보하여 가독성을 높였습니다.

네비게이션 개선

기존의 복잡한 메뉴를 햄버거 메뉴로 변경하고, 중요도가 높은 메뉴를 상단에 배치하여 접근성을 높였습니다. 터치 영역을 충분히 확보하여 오클릭을 방지했습니다.

이미지 및 자원 최적화

반응형 이미지를 적용하고(srcset), 웹팩(Webpack)을 사용하여 이미지와 자바스크립트 파일을 압축, 번들링하여 초기 로딩 시간을 단축했습니다. 지연 로딩(Lazy Loading)을 구현하여 불필요한 자원 로딩을 최소화했습니다.

결과 & 느낀 점

UX 개선 프로젝트를 통해 모바일 환경에서의 이탈률을 15%로 감소시켰으며, 사용자 만족도를 20% 향상시켰습니다. 또한, 페이지 로딩 속도를 평균 2초 단축하는 성과를 달성했습니다.

이번 경험을 통해 문제점을 데이터 기반으로 분석하고, 사용자 중심의 해결 방안을 모색하며, 기술적으로 이를 구현하는 역량을 키울 수 있었습니다. 사용자 피드백을 통해 끊임없이 개선해나가는 개발의 중요성을 깨달았습니다.

프로젝트: 디저트 쇼핑몰

Q&A(질문과 답변)를 활용한 웹사이트 구축 프로젝트의 일환으로, 고객 문의 및 리뷰 시스템을 갖춘 디저트 쇼핑몰 웹사이트를 개발했습니다. 기획부터 디자인, 프론트엔드 및 백엔드 개발, 데이터베이스 구축까지 프로젝트의 전반적인 과정을 팀원들과 함께 수행했습니다.

기술 스택: React, Node.js (Express), MySQL, HTML, CSS

개인 기여도

  • 기획 및 디자인 (80%): 사용자 피드백을 기반으로 쇼핑몰 기능을 기획하고, Figma를 활용하여 직관적인 레이아웃과 UI/UX 디자인을 담당했습니다.
  • 프론트엔드 개발 (100%): React를 사용하여 상품 목록, 상세 페이지, 장바구니, Q&A 게시판 등 사용자 인터페이스를 구현했습니다.
  • 백엔드 개발 (10%): Node.js와 Express를 활용하여 상품 정보, 주문, Q&A 게시글을 관리하는 RESTful API를 설계 및 개발했습니다.
  • 데이터베이스 설계 (10%): MySQL을 사용하여 상품, 주문, 회원, Q&A 등 필요한 모든 테이블 스키마를 설계했습니다.

개발 과정 & 문제 해결

  • **개발 프로세스**: 기획(요구사항 정의) → 디자인(Figma) → 프론트엔드/백엔드 동시 개발 → 통합 테스트
  • **기술적 도전**: 고객 Q&A 게시판에서 이미지 업로드 및 실시간으로 댓글이 추가되는 기능을 구현하는 데 어려움이 있었습니다.
  • **해결 방안**:
    1. **이미지 업로드**: Multer 라이브러리를 사용하여 서버에 이미지를 안전하게 저장하고, 클라이언트에서는 이미지 경로를 전송하여 처리했습니다.
    2. **실시간 업데이트**: 웹소켓 대신 React의 상태 관리와 API 폴링 방식을 사용하여 댓글이 추가되거나 삭제될 때마다 해당 컴포넌트를 자동으로 리렌더링하도록 구현하여 실시간성을 확보했습니다.

결과 & 느낀 점

사용자 친화적인 UI와 효율적인 Q&A 시스템을 갖춘 디저트 쇼핑몰 웹사이트를 성공적으로 완성했습니다.

**배운 점**: 처음부터 끝까지 하나의 서비스를 완성해보는 경험을 통해, 프론트엔드와 백엔드 간의 데이터 흐름을 명확하게 이해할 수 있었습니다. 특히, 고객의 질문과 피드백을 처리하는 Q&A 기능을 구현하면서 사용자 중심의 기능 설계와 데이터 관리의 중요성을 깨달았습니다. 팀 프로젝트를 통해 협업의 가치와 함께 문제를 해결하는 과정을 경험할 수 있었던 의미 있는 시간이었습니다.
